## О проекте

**Распознавание рукописных цифр с дообучением** — это полноценное desktop-приложение на Python, которое позволяет:

- Рисовать цифры мышью на интерактивном холсте
- Мгновенно распознавать их с помощью свёрточной нейросети
- **Исправлять ошибки модели** — если нейросеть ошиблась, пользователь указывает правильную цифру
- **Автоматически дообучать модель** на собранных исправлениях — каждые 50 новых примеров модель становится умнее

Проект демонстрирует полный цикл ML-разработки: от сбора данных и обучения модели до создания пользовательского интерфейса и реализации системы обратной связи.

**Это не просто "распознавалка цифр". Это система, которая учится на своих ошибках и становится лучше с каждым использованием.**

---

## Проблема и мотивация

В реальной жизни ML-модели не живут в вакууме. Данные меняются, появляются новые паттерны, и то, что работало вчера, сегодня может ошибаться.

**Основная проблема:** большинство ML-проектов умирают после первой демонстрации. Модель показала 98% точности на тестовом датасете — и всё, дальше она не развивается. Никто не собирает её ошибки, никто не адаптирует под реальных пользователей.

**Моя мотивация:** создать систему, которая:
1. Используется в реальных условиях (не в Jupyter Notebook)
2. Собирает обратную связь от пользователей
3. Автоматически улучшается на основе этой обратной связи
4. Становится умнее с каждым днём использования

## Решение

Я разработал приложение с **замкнутым циклом обучения**:

```
Пользователь рисует цифру
       ↓
Модель распознаёт
       ↓
  ┌────┴────┐
  │  Правильно? │
  └────┬────┘
  Да     Нет
   ↓      ↓
Показ   Пользователь
результата выбирает
         ↓
     Сохранение
     ошибки
         ↓
    Накопление
    50 ошибок
         ↓
   Дообучение
    модели
         ↓
   Модель стала
     умнее!
```

**Ключевая идея:** пользователь не просто потребляет результат — он **участвует в обучении модели**. Чем больше людей используют приложение, тем умнее становится нейросеть.

---

## Ключевая фишка: обучение на ошибках

Это главное, что отличает этот проект от тысяч других "распознавалок цифр".

### Как это работает:

| Шаг | Действие | Что происходит |
|-----|----------|----------------|
| 1 | Пользователь рисует цифру | Холст 400×400, белая кисть на чёрном фоне |
| 2 | Нажимает "Распознать" | Модель предсказывает цифру + уверенность |
| 3 | Модель говорит: "Это 2" | Например, хотя нарисована 7 |
| 4 | Пользователь видит ошибку | Нажимает кнопку "Неверно" |
| 5 | Появляются кнопки 0-9 | Пользователь выбирает правильную цифру |
| 6 | Изображение сохраняется | В папку `corrections/7/123.png` |
| 7 | Счётчик ошибок растёт | Каждые 50 ошибок — дообучение |
| 8 | Модель переобучается | 20 эпох на новых данных |
| 9 | Модель сохраняется | `mnist_cnn.keras` обновлена |
| 10 | Пользователь видит уведомление | "Модель обновлена! Спасибо!" |

### Почему это работает:

1. **Сбор данных происходит автоматически** — пользователи не тратят время на разметку
2. **Модель адаптируется под реальный стиль рисования** — не под абстрактный MNIST
3. **Эффект масштабирования** — чем больше пользователей, тем лучше модель
4. **Обучение с подкреплением** — модель учится на своих ошибках, как человек

### preprocess.py — подготовка изображений

Этот модуль делает "грязную работу" — превращает рисунок пользователя в формат, понятный нейросети.

**Шаг 1: Бинаризация**
```python
gray = cv2.cvtColor(img_array, cv2.COLOR_BGR2GRAY)
_, binary = cv2.threshold(gray, 127, 255, cv2.THRESH_BINARY)
```

**Шаг 2: Обрезка по контуру**
```python
coords = cv2.findNonZero(binary)
x, y, w, h = cv2.boundingRect(coords)
margin = 4
digit = binary[y-margin:y+h+margin, x-margin:x+w+margin]
```

**Шаг 3: Ресайз с сохранением пропорций**
```python
result = np.zeros((28, 28), dtype=np.uint8)
if w > h:
    new_w = 20
    new_h = int(h * (20 / w))
else:
    new_h = 20
    new_w = int(w * (20 / h))
resized = cv2.resize(digit, (new_w, new_h))
start_x = (28 - new_w) // 2
start_y = (28 - new_h) // 2
result[start_y:start_y+new_h, start_x:start_x+new_w] = resized
```

**Шаг 4: Нормализация**
```python
result = result.astype('float32') / 255.0
result = np.expand_dims(result, axis=(0, 3))
```

## Препроцессинг изображений

Это один из самых важных компонентов проекта. Без правильного препроцессинга модель не сможет работать.

### Почему препроцессинг важен:

| Проблема | Решение |
|----------|---------|
| Рисунок 400×400 | Ресайз до 28×28 |
| Фон может быть неровным | Бинаризация |
| Цифра может быть со смещением | Обрезка по контуру + центрирование |
| Разные пропорции цифр | Ресайз с сохранением пропорций |
| Пиксели 0-255 | Нормализация к [0, 1] |

### Этапы препроцессинга в картинках:

```
Исходный рисунок (400×400)
         ↓
Бинаризация (чёрно-белое)
         ↓
Обрезка по контуру (убрать пустые поля)
         ↓
Ресайз до 28×28 с сохранением пропорций
         ↓
Центрирование (вставить в центр)
         ↓
Нормализация (деление на 255)
         ↓
Готово для модели (28×28, [0, 1])
```

## Модель

### Архитектура (LeNet-5)

Модель основана на классической архитектуре LeNet-5, адаптированной для MNIST.

```python
model = keras.Sequential([
    Conv2D(32, (3,3), activation='relu', input_shape=(28,28,1)),
    MaxPooling2D(2,2),
    Conv2D(64, (3,3), activation='relu'),
    MaxPooling2D(2,2),
    Conv2D(128, (3,3), activation='relu'),
    Flatten(),
    Dense(128, activation='relu'),
    Dropout(0.5),
    Dense(10, activation='softmax')
])
```

### Почему такая архитектура:

| Слой | Назначение |
|------|------------|
| Conv2D + ReLU | Извлечение признаков (углы, линии, кривые) |
| MaxPooling2D | Уменьшение размерности, устойчивость к смещениям |
| Flatten | Преобразование 2D в 1D |
| Dense + ReLU | Обучение сложным комбинациям признаков |
| Dropout | Предотвращение переобучения |
| Dense + Softmax | Классификация (10 классов) |

### Обучение:

**Датасет:** MNIST (60 000 тренировочных, 10 000 тестовых)  
**Эпохи:** 20  
**Batch size:** 32  
**Точность:** 99.2% на тестовой выборке

## Дообучение

### Алгоритм дообучения:

1. **Сбор исправлений**
   - Пользователь нажимает "Неверно"
   - Выбирает правильную цифру
   - Изображение сохраняется в `corrections/{цифра}/`

2. **Периодическое дообучение**
   - Каждые 50 новых исправлений запускается `retrain.py`
   - Модель дообучается на всех накопленных исправлениях
   - 20 эпох, learning rate 0.0001 (тонкая настройка)

3. **Сохранение**
   - Обновлённая модель сохраняется как `mnist_final.keras`
   - При следующем запуске используется новая версия

### Почему 50 исправлений:

- Слишком мало (< 10) — модель запомнит шум
- Слишком много (> 500) — модель не успевает адаптироваться
- 50 — оптимальный баланс между скоростью и качеством

## Безопасность

### Текущее состояние:

| Аспект | Статус |
|--------|--------|
| SQL-инъекции |  Защищено (параметризованные запросы) |
| Пароли |  Хранятся в открытом виде |


### Планы по безопасности:

1. **Хеширование паролей** — использовать bcrypt для хранения паролей
2. **JWT токены** — для сессий пользователей
3. **Переменные окружения** — для секретов (не хранить в коде)

### Почему это важно:

В учебном проекте допустимо хранить пароли в открытом виде. Но если проект идёт в продакшен — это критическая уязвимость.
